Getting Started

MockFactory provides cloud emulation, code execution sandboxes, and mock infrastructure for testing. Get started in minutes.

1. Get Your API Key

  1. 1. Sign up at mockfactory.io/app
  2. 2. Get $5 free credits automatically
  3. 3. Generate an API key from your dashboard
  4. 4. Copy your API key (starts with mf_)
# Set as environment variable
export MOCKFACTORY_API_KEY="mf_your_key_here"

MockFactory CLI

Secure code execution sandbox and mock resource management from your terminal.

Installation

pip install mockfactory-cli
‚Üí View on PyPI

Basic Usage

# Execute Python code
mockfactory run python -c "print('Hello World')"
# Create mock user
mockfactory user create john.doe --email john@example.com
# Create organization
mockfactory organization create acme-corp --plan pro

Code Execution

Execute code in secure, isolated containers

# Execute inline code
mockfactory run python -c "import sys; print(sys.version)"
mockfactory run javascript -c "console.log('Hello')"
mockfactory run php -c "echo phpversion();"
# Execute from file
mockfactory execute script.py
mockfactory execute app.js
# With custom timeout
mockfactory execute long_task.py --timeout 120

Organizations & Hierarchy

Create organizational structures with domains, clouds, and projects

# Create organization
mockfactory organization create acme-corp \
--description "Acme Corporation" \
--plan pro
# Create domain
mockfactory domain create example.com \
--organization acme-corp \
--verified
# Create cloud environment
mockfactory cloud create dev-cloud \
--provider aws \
--organization acme-corp \
--region us-east-1
# Create project
mockfactory project create web-app \
--organization acme-corp \
--environment production

Users & Groups

Manage mock users and groups for testing

# Create user
mockfactory user create john.doe \
--email john@example.com \
--full-name "John Doe" \
--role developer \
--organization acme-corp \
--cloud dev-cloud
# Create group
mockfactory group create developers \
--description "Development team"
# Add user to group
mockfactory group add-user developers john.doe
# List users
mockfactory user list --role admin
mockfactory user list --organization acme-corp

Containers & Networks

Create mock containers and networks for testing

# Create container
mockfactory container create web-app \
--image nginx \
--user john.doe \
--network frontend
# Create network
mockfactory network create frontend \
--cidr 10.1.0.0/24 \
--isolated
# Bind user to container
mockfactory container bind-user web-app alice

Policies & Roles

# Python SDK
policy = mf.iam.create_policy(
    policy_name="s3-read-only",
    policy_document={
        "Version": "2012-10-17",
        "Statement": [{
            "Effect": "Allow",
            "Action": ["s3:GetObject", "s3:ListBucket"],
            "Resource": "*"
        }]
    }
)

role = mf.iam.create_role(
    role_name="lambda-execution",
    trust_policy={
        "Service": "lambda.amazonaws.com"
    }
)

Permission Checks

# CLI
mockfactory iam check-permission john.smith \
  --action s3:GetObject \
  --resource my-bucket/file.txt

mockfactory iam simulate-policy s3-read-only \
  --action s3:GetObject \
  --resource bucket/key

# Python SDK
result = mf.iam.check_permission(
    username="john.smith",
    action="s3:GetObject",
    resource="my-bucket/file.txt"
)
print(f"Allowed: {result['allowed']}")

# Simulate policy
result = mf.iam.simulate_policy(
    policy_name="s3-read-only",
    action="s3:GetObject",
    resource="bucket/key"
)

API Reference

Complete REST API reference for programmatic access to MockFactory.

Authentication

All API requests require authentication using your API key in the Authorization header: 

Authorization: Bearer mf_your_api_key_here

API Base URL

https://api.mockfactory.io/v1

Rate Limits

Plan Requests/Minute Burst
Free 60 100
Pro 300 500
Enterprise Unlimited Unlimited

Rate limit headers are returned with each response: 

X-RateLimit-Limit: 60
X-RateLimit-Remaining: 58
X-RateLimit-Reset: 1640995200

Error Handling

MockFactory uses conventional HTTP response codes:

Code Description
200 Success
400 Bad Request - Invalid parameters
401 Unauthorized - Invalid API key
429 Too Many Requests - Rate limit exceeded
500 Internal Server Error

Error response format: 

{
  "error": {
    "code": "invalid_parameter",
    "message": "cidr_block is required",
    "param": "cidr_block"
  }
}
